## рҹ”Ң Tabla de Conectividad Detallada

### Conexiones de Usuarios вҶ’ Dashboard

| Origen | Destino | Puerto | Protocolo | DirecciГіn | Observaciones |
|--------|---------|--------|-----------|-----------|---------------|
| Navegador Usuario | Servidor Dashboard | 80 | HTTP | Entrada | TrГЎfico web no encriptado |
| Navegador Usuario | Servidor Dashboard | 443 | HTTPS | Entrada | TrГЎfico web encriptado (recomendado) |

**ConfiguraciГіn de Firewall (Dashboard Server):**
```bash
# Permitir entrada HTTP/HTTPS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# O con iptables:
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT
```

### Conexiones Dashboard вҶ’ Dominios WebLogic

| Origen | Destino | Puerto | Protocolo | DirecciГіn | Observaciones |
|--------|---------|--------|-----------|-----------|---------------|
| Backend (FastAPI) | AdminServer Dominio 1 | 7001 | T3 | Salida | Consulta de mГ©tricas |
| Backend (FastAPI) | AdminServer Dominio 2 | 7001 | T3 | Salida | Consulta de mГ©tricas |
| Backend (FastAPI) | AdminServer Dominio 3 | 7001 | T3 | Salida | Consulta de mГ©tricas |
| ... | ... | ... | ... | ... | ... |
| Backend (FastAPI) | AdminServer Dominio 90 | 7001 | T3 | Salida | Consulta de mГ©tricas |

**Notas:**
- El puerto puede ser diferente (7001, 7002, 8001, etc.)
- Si usas SSL: puerto 7002 con protocolo T3S
- Frecuencia: Cada 15 segundos (configurable)

**ConfiguraciГіn de Firewall (Dashboard Server - Salida):**
```bash
# Permitir salida a puerto 7001
sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" port protocol="tcp" port="7001" accept'
sudo firewall-cmd --reload

# O con iptables:
sudo iptables -A OUTPUT -p tcp --dport 7001 -j ACCEPT
```

**ConfiguraciГіn de Firewall (Servidores WebLogic - Entrada):**
```bash
# En cada servidor WebLogic, permitir desde IP del Dashboard
# Reemplazar 192.168.1.100 con la IP real del servidor Dashboard

sudo firewall-cmd --permanent --add-rich-rule='rule family="ipv4" source address="192.168.1.100" port protocol="tcp" port="7001" accept'
sudo firewall-cmd --reload

# O con iptables:
sudo iptables -A INPUT -s 192.168.1.100 -p tcp --dport 7001 -j ACCEPT
```

### Consumo de Red Esperado

**Por Dominio:**
- Request WLST: ~5-10 KB
- Response con mГ©tricas: ~20-50 KB

**Para 90 Dominios:**
- Primera consulta (sin cache): ~2.7-5.4 MB
- Consultas subsecuentes (con cache): ~0 MB (no hay trГЎfico de red)
- Cada 15 segundos: Consulta a ~6 dominios (rotaciГіn de cache)

**TrГЎfico Total Estimado:**
- Por hora: ~60-120 MB
- Por dГӯa: ~1.5-3 GB

| Escenario | Tiempo |

## рҹҡЁ Problemas Comunes de Conectividad

### Problema: "No route to host"

**Causa:** Firewall bloqueando  
**SoluciГіn:** Configurar firewall en servidor Dashboard y WebLogic

### Problema: "Connection timeout"

**Causa:** AdminServer no escuchando en puerto o firewall intermedio  
**SoluciГіn:** Verificar AdminServer corriendo y firewall de red

### Problema: "Connection refused"

**Causa:** Servicio no corriendo en el puerto  
**SoluciГіn:** Verificar que AdminServer estГ© UP

### Problema: "Name resolution failed"

**Causa:** DNS no configurado  
**SoluciГіn:** Agregar entradas a /etc/hosts o configurar DNS
